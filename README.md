## 📌 Overview

This repository documents my professional experience as a **QA Engineer** working on the Neolude LMS platform between **February 2019 and May 2024**.

During this period, I was responsible for ensuring that new features were delivered with quality, stability, and alignment with business requirements, while preserving system performance and user experience.

---

## 🎯 Objective

Ensure that all new features implemented in the LMS:

- Met functional requirements  
- Did not introduce regressions  
- Maintained system integrity  
- Preserved user experience  
- Operated with stability and acceptable performance  

The role involved continuous validation across development, staging, and production support environments.

---

## 🧪 Testing Strategy

### 🔍 Functional Testing
Validation of business rules and compliance with specified requirements.

### 🧠 Exploratory Testing
Structured exploration of the system to identify unexpected defects and edge-case behaviors.

### 🔁 Regression Testing
Recurrent execution of test scenarios to ensure new releases did not impact existing features.

### 🔗 Integration Testing
Validation of interactions between internal modules and external system integrations.

### 🐞 Bug Fix Validation
Retesting of reported defects to ensure proper resolution without side effects.

### 🤖 Test Automation
Automation of critical scenarios using:

- Selenium  
- Cypress  

Focus areas:

- Automated regression coverage  
- Core user flows  
- Execution efficiency and repeatability

## 🏗️ Product Context

Neolude is a Learning Management System (LMS) used by companies and institutions to:

- Manage and distribute courses  
- Apply gamification strategies  
- Monitor learner progress  
- Generate performance reports  
- Integrate with corporate systems (e.g., HR platforms)  
- Provide cross-platform access (desktop and mobile)  
- Deliver personalized learning paths  

The system includes multiple user roles, permissions, integrations, and complex business rules.
